# 安全策略

## 报告安全漏洞

如果您发现 Multi-Business CRM 中的安全漏洞，请**不要**通过公开的 GitHub Issue 报告。

### 报告方式

请通过以下方式私下报告安全问题：

1. **电子邮件**：发送至项目维护者的私人邮箱（在 GitHub 个人资料中可找到）
2. **GitHub Security Advisories**：使用 GitHub 的私密安全通告功能

### 报告内容

请包含以下信息：

- 漏洞的详细描述
- 复现步骤
- 潜在影响范围
- 建议的修复方案（如有）
- 您的联系方式（以便后续沟通）

### 响应时间

- **确认收到**：48 小时内
- **初步评估**：5 个工作日内
- **修复计划**：根据严重程度，1-30 天内
- **严重漏洞**：优先处理，可能在 7 天内修复

## 支持的版本

| 版本 | 支持状态 |
|------|----------|
| 0.1.x | 当前版本，完全支持 |

## 安全更新

安全修复将发布在：

- `CHANGELOG.md` 中以 **[SECURITY]** 标签标注
- GitHub Security Advisories
- 新的 Git Tag（如 `v0.1.1`）

## 安全最佳实践

### 生产部署

- 始终通过 HTTPS 提供服务
- 设置强随机 `JWT_SECRET`
- 使用最新的稳定版本
- 定期执行 `npm audit` 检查依赖漏洞
- 定期备份数据库

### 开发环境

- 不在代码中硬编码密钥
- 使用 `.env.local` 覆盖本地配置
- 不将 `.env` 文件提交到版本控制
- 开发完成后清除测试数据
