# 系统设置

## 概述

系统设置是 CRM 的全局配置中心，提供 AI 配置、AI 控制面板、IM 集成、邮件管理、业务线管理、模板管理、外部数据源、系统健康检查和操作日志等功能的入口和配置。不同角色的用户拥有不同的设置访问权限。

## 页面入口

| 页面 | 路由 | 说明 |
|------|------|------|
| 系统设置 | `/settings` | 设置主页面（导航入口） |

## 子模块

### AI 配置

**路由：** `/ai-settings`

管理 AI 服务的相关配置，包括 AI 提供商选择、API 密钥设置和模型选择。详见 [AI 功能](./11-ai-features.md) 章节。

### AI 控制面板

**路由：** `/ai-control-panel`

管理 AI 功能的全局开关、模块开关、执行模式、策略规则和执行日志。详见 [AI 控制面板](./12-ai-control-panel.md) 章节。

### IM 设置

**路由：** `/im-settings`

配置飞书（Feishu/Lark）平台的集成，用于消息通知和沟通协作。

### 邮件账户管理

**路由：** `/email/accounts`

管理邮件中心的邮箱账户绑定和配置。详见 [邮件中心](./08-email.md) 章节。

### 业务线管理

**路由：** `/business-lines`

定义和管理企业的业务线。

#### 业务线字段（BusinessLine）

| 字段 | 类型 | 说明 |
|------|------|------|
| name | String | 业务线名称（唯一） |
| code | String | 业务线代码（唯一） |
| description | String | 业务线描述 |
| website | String | 业务线相关网站 |
| mainProducts | String | 主要产品 |

#### 业务线关联

业务线可关联以下实体：

- 线索（Leads）
- 客户（Customers）
- 项目（Projects）
- 产品（Products）
- 订单（Orders）
- 模板（Templates）
- 外部数据源（External Sources）

### 模板管理

**路由：** `/templates`

管理跟进沟通模板，支持按业务场景和语言分类。

### 外部数据源

**路由：** `/external-sources`

配置 Webhook 来源，用于外部系统自动导入线索数据。

### 系统健康检查

**路由：** `/system-health`

查看系统运行状态、数据库连接、API 可用性等健康指标。

### 操作日志

**路由：** `/activity-logs`

查看系统操作审计日志，记录所有用户操作行为。

## 用户角色（Role）

系统支持三种用户角色，不同角色拥有不同的功能访问权限：

| 角色 | 说明 | 权限范围 |
|------|------|------|
| ADMIN（管理员） | 系统最高权限用户 | 所有功能和设置的完全访问权限 |
| MANAGER（经理） | 管理层用户 | 业务数据查看、团队管理、报表访问 |
| SALES（销售） | 普通销售人员 | 日常业务操作（客户、线索、报价等） |

### 权限说明

- **ADMIN：** 可访问所有设置页面，包括 AI 配置、业务线管理、系统健康检查和操作日志。
- **MANAGER：** 可查看报表和团队数据，可管理业务线和模板。
- **SALES：** 可执行日常业务操作，但无法修改系统级配置。

## 使用说明

### 访问系统设置

1. 在系统导航栏中找到并点击「设置」入口。
2. 在设置主页面中选择需要配置的子模块。

### 修改 AI 配置

1. 进入 AI 配置页面（`/ai-settings`）。
2. 选择 AI 提供商，填写 API 密钥和模型信息。
3. 保存配置后可使用 AI 测试页面验证。

### 管理业务线

1. 进入业务线管理页面（`/business-lines`）。
2. 点击「新建业务线」。
3. 填写业务线名称（唯一）、代码（唯一）、描述、网站和主要产品。
4. 保存业务线信息。
5. 后续可在创建线索、客户等实体时选择所属业务线。

### 查看系统健康状态

1. 进入系统健康检查页面（`/system-health`）。
2. 查看数据库连接状态、API 响应时间、磁盘空间等指标。
3. 如发现异常，及时联系系统管理员。

### 查看操作日志

1. 进入操作日志页面（`/activity-logs`）。
2. 按时间、用户、操作类型等条件筛选日志。
3. 查看操作详情，用于审计和问题排查。

## 注意事项

- 系统设置的修改权限受用户角色限制，普通销售人员无法访问系统级配置。
- AI 配置涉及 API 密钥，请注意信息安全，不要在日志中暴露密钥信息。
- 业务线名称和代码一旦创建不可修改（唯一约束），创建时请谨慎填写。
- 建议由 ADMIN 角色用户进行系统级配置，避免误操作影响系统运行。
- 完整的多租户功能将在 v2.0 版本中提供，当前版本部分支持租户基础信息管理。
