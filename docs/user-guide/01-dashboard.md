# 工作台与仪表盘

## 概述

工作台（Workbench）和仪表盘（Dashboard）是系统的核心入口，提供日常工作的概览视图和业务数据分析能力。访问系统根路径 `/` 将自动跳转至工作台。

---

## 工作台（Workbench）

**访问地址：** `/workbench`

工作台是业务人员每日登录后的主页面，集中展示待办事项、日程安排和关键业务指标。

### 日历组件（ScheduleWidget）

页面顶部区域显示当日日历小部件（ScheduleWidget 组件），以时间线形式展示今天的日程安排，包括：

- 计划中的跟进活动
- 已安排的会议
- 系统自动创建的任务提醒

### 任务列表

- 显示今日待办任务
- **逾期任务**会以高亮样式标记，提醒用户优先处理
- 支持直接点击任务跳转到任务详情页

### 快捷操作按钮

提供常用功能的快速入口，减少页面导航步骤。包括：

- 新建线索（Lead）
- 新建客户（Customer）
- 新建跟进记录（FollowUp）
- 新建任务（Task）

### 今日跟进提醒

集中展示今天需要跟进的线索和客户，帮助业务人员合理安排工作优先级。

### 右侧面板（RightPanel）

页面右侧展示综合数据面板（`RightPanel.tsx` 组件，宽度 320px，屏幕宽度不足时自动隐藏），包含以下模块：

| 模块 | 说明 |
|------|------|
| **销售目标** | 以进度条形式展示各指标的完成情况，支持货币格式化显示，点击可跳转至 `/goals` 目标管理页面 |
| **任务统计** | 显示已完成、逾期、待处理的任务数量统计 |
| **项目状态分布** | 基于 Recharts 组件渲染的饼图（Pie Chart），直观展示各阶段项目的分布情况 |
| **邮件统计** | 显示已发送和已接收的邮件数量 |

---

## 仪表盘（Dashboard）

**访问地址：** `/dashboard`

仪表盘提供全局业务数据的可视化分析，帮助管理者了解业务全局状况。

### 转化漏斗图

以漏斗图形式展示线索从初始状态到成交的转化过程：

```
线索（Leads）→ 合格线索（Qualified）→ 成交（Won）
```

直观呈现各阶段的转化率和流失情况。

### 营收统计

汇总展示各业务线的营收数据，包括：

- 按时间维度的收入趋势
- 按业务线的收入对比
- 按币种（USD/EUR/CNY）的收入分布

### AI 智能洞察与建议

系统基于 AI 分析引擎，自动生成业务洞察：

- 线索质量评分分析
- 高潜力客户推荐
- 跟进优先级建议
- 业务趋势预测

> **当前版本部分支持**：AI 洞察功能依赖于 AI 服务配置（需要在 `.env` 中设置 `AI_API_KEY`、`AI_BASE_URL`、`AI_MODEL`），未配置时该模块不显示数据。

### 线索来源分布

以图表形式展示各线索来源的占比情况，来源包括：

- 网站（WEBSITE）
- Facebook
- TikTok
- WhatsApp
- 邮件（EMAIL）
- 展会（EXHIBITION）
- 介绍（REFERRAL）
- 人工开发（MANUAL_OUTREACH）
- 其他（OTHER）

### 业务线业绩对比

对比不同业务线（BusinessLine）的核心指标，支持横向比较各业务线的表现差异。

---

## 注意事项

1. 工作台右侧面板在小屏幕设备（宽度低于 `xl`）上会自动隐藏。
2. AI 相关功能需要先配置 AI 服务环境变量，详见系统部署文档。
3. 数据统计均基于实时查询，页面刷新后数据会同步更新。
4. 多租户（Multi-tenant）数据隔离功能当前版本部分支持，计划在 v2.0 版本中全面实现。
