# 跟进记录与任务管理（Follow-ups & Tasks）

## 概述

跟进记录（Follow-up）和任务（Task）是外贸业务日常管理的核心工具。跟进记录用于记录每次与客户或线索的沟通详情，任务则用于管理和追踪待办事项。两者紧密配合，确保业务跟进不遗漏。

## 跟进记录（FollowUp）

### 主要字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `method` | 枚举 (FollowUpMethod) | 跟进方式 |
| `content` | 文本 | 跟进内容 |
| `customerFeedback` | 文本 | 客户反馈 |
| `nextAction` | 文本 | 下一步行动 |
| `followUpDate` | 日期 | 跟进日期 |
| `nextFollowUpDate` | 日期 | 下次跟进日期 |
| `remark` | 文本 | 备注 |

### 关联字段（可选）

| 字段 | 类型 | 说明 |
|------|------|------|
| `leadId` | 关联 | 关联线索 ID |
| `customerId` | 关联 | 关联客户 ID |
| `projectId` | 关联 | 关联项目 ID |

### FollowUpMethod 跟进方式枚举值

| 值 | 中文 | 说明 |
|----|------|------|
| `EMAIL` | 邮件 | 通过电子邮件沟通 |
| `WHATSAPP` | WhatsApp | 通过 WhatsApp 沟通 |
| `PHONE` | 电话 | 通过电话沟通 |
| `MEETING` | 会议 | 面对面或线上会议 |
| `VIDEO_CALL` | 视频通话 | 通过视频通话沟通 |
| `OTHER` | 其他 | 其他沟通方式 |

### 操作步骤

#### 新建跟进记录

1. 导航至 `/follow-ups/new`，或在线索/客户详情页点击"添加跟进"
2. 选择跟进方式
3. 填写跟进内容和客户反馈
4. 设定跟进日期和下次跟进日期
5. 关联对应的线索、客户或项目（可选）
6. 点击"保存"完成创建

#### 编辑跟进记录

1. 在跟进详情页点击"编辑"按钮
2. 修改需要更新的字段
3. 点击"保存"完成编辑

### 关联字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `leadId` | 关联 | 关联线索 ID |
| `customerId` | 关联 | 关联客户 ID |

| `quoteId` | 关联 | 关联报价单 ID |

### TaskType 任务类型枚举值

| 值 | 中文 | 说明 |
|----|------|------|
| `FOLLOW_UP` | 跟进 | 跟进客户或线索 |
| `CALL` | 电话 | 拨打电话 |
| `MEETING` | 会议 | 安排或参加会议 |
| `QUOTE` | 报价 | 准备或发送报价 |
| `SAMPLE` | 寄样 | 安排样品寄送 |
| `OTHER` | 其他 | 其他类型任务 |

### TaskStatus 任务状态枚举值

| 值 | 中文 | 说明 |
|----|------|------|
| `PENDING` | 待处理 | 任务尚未开始 |
| `IN_PROGRESS` | 进行中 | 任务正在进行 |
| `COMPLETED` | 已完成 | 任务已完成 |
| `CANCELLED` | 已取消 | 任务已取消 |

### TaskPriority 任务优先级枚举值

| 值 | 中文 | 说明 |
|----|------|------|
| `LOW` | 低 | 不紧急的任务 |
| `MEDIUM` | 中 | 一般优先级 |
| `HIGH` | 高 | 需要优先处理 |
| `URGENT` | 紧急 | 必须立即处理 |

### 任务状态流转

```
PENDING（待处理）
  ↓
IN_PROGRESS（进行中）
  ↓
COMPLETED（已完成）

PENDING / IN_PROGRESS → CANCELLED（已取消）
```

### 操作步骤

#### 新建任务

1. 导航至 `/tasks/new`，或在工作台点击"新建任务"快捷按钮
2. 填写任务标题和描述
3. 选择任务类型和优先级
4. 设定截止日期
5. 关联对应的线索、客户、项目、报价单或订单（可选）
6. 点击"保存"完成创建

#### 完成任务

1. 在任务详情页，将状态变更为"已完成"
2. 系统自动记录完成时间（`completedAt`）

## 自动任务创建

系统通过事件总线（Event Bus）在关键业务事件发生时自动创建任务，减少人工操作负担。

### 事件触发的自动任务

| 触发事件 | 自动创建的任务 | 优先级 | 截止时间 |
|----------|---------------|--------|----------|
| 新线索创建（`lead.created`） | "跟进新线索：{公司名称}" | 高（HIGH） | 次日 09:00 |
| 报价已发送（`quote.sent`） | "跟进报价：{报价编号}" | 中（MEDIUM） | 次日 10:00 |
| 订单已确认（`order.confirmed`） | "跟进生产进度：{订单编号}" | 中（MEDIUM） | 7 天后 09:00 |

### AI 跟进代理（Follow-up Agent）

系统内置 AI 跟进代理（`followup-agent.ts`），定期自动检测需要跟进的线索和客户：

**检测规则 1：休眠线索**
- 检测条件：线索状态为 `NEW` 或 `CONTACTED`，且超过 **3 天**未有任何跟进记录
- 自动创建任务："AI 自动跟进：{公司名称}"
- 优先级：中（MEDIUM）
- 每次检测最多处理 10 条线索

**检测规则 2：不活跃客户**
- 检测条件：客户状态为 `ACTIVE`，且超过 **7 天**未有任何跟进记录
- 自动创建任务："AI 客户回访提醒：{公司名称}"
- 优先级：高（HIGH）
- 每次检测最多处理 10 个客户

> **注意：** AI 跟进代理会检查是否已存在相同标题的待处理任务，避免重复创建。

## 逾期检测

在工作台（Workbench）中，系统会自动高亮显示逾期任务：

- **逾期条件**：任务的 `dueDate` 早于当前时间，且状态不是 `COMPLETED`
- **展示方式**：逾期任务在任务列表中以醒目的样式标记
- **处理建议**：优先处理逾期任务，或根据实际情况调整截止日期

## 已知限制

1. **自动任务触发**：自动任务仅在以下三个事件时创建，暂不支持自定义触发规则。
2. **AI 跟进代理检测频率**：AI 跟进代理的运行频率由系统控制，当前版本部分支持自定义检测周期。
3. **任务依赖**：任务之间暂不支持设置依赖关系（如前置任务必须完成后才能开始后续任务）。
4. **批量操作**：批量完成或批量修改任务状态的功能当前版本部分支持。
