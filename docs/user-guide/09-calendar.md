# 日历

## 概述

日历模块用于管理外贸业务中的各类日程安排，包括任务、会议、节假日提醒和跟进计划。日历事件可与客户、线索、项目关联，帮助团队高效管理日常工作。工作台的"今日日程"小组件（ScheduleWidget）会展示当天的所有日程事件。

## 页面入口

| 页面 | 路由 | 说明 |
|------|------|------|
| 日历 | `/calendar` | 日历主页面 |

## API 端点

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/calendar-events` | 获取日历事件列表 |
| POST | `/api/calendar-events` | 创建日历事件 |

## 日历事件字段（CalendarEvent）

| 字段 | 类型 | 说明 |
|------|------|------|
| title | String | 事件标题 |
| description | String | 事件描述 |
| eventType | Enum | 事件类型：`task`（任务）/ `meeting`（会议）/ `holiday`（节假日）/ `reminder`（提醒） |
| startTime | DateTime | 开始时间 |
| endTime | DateTime | 结束时间 |
| allDay | Boolean | 是否为全天事件 |
| isCompleted | Boolean | 是否已完成 |
| customerId | String | 关联的客户 ID（可选） |
| leadId | String | 关联的线索 ID（可选） |
| projectId | String | 关联的项目 ID（可选） |

## 视图模式

日历提供三种视图模式，满足不同使用场景：

| 视图 | 说明 |
|------|------|
| 周视图（Week） | 以一周为周期展示日程，适合查看近期安排 |
| 月视图（Month） | 以整月为周期展示日程，适合宏观规划 |
| 列表视图（List） | 以时间列表形式展示所有事件，适合查看详情 |

## 事件来源

日历事件通过以下方式自动或手动创建：

| 来源 | 说明 |
|------|------|
| 任务（Task） | 创建或更新任务时自动生成对应日历事件 |
| 跟进记录（FollowUp） | 创建跟进计划时自动生成日历事件 |
| 手动创建 | 在日历页面直接创建自定义事件 |

## 关联实体

日历事件可关联以下 CRM 实体：

- **客户（Customer）** — 与客户相关的会议或跟进日程。
- **线索（Lead）** — 与线索相关的跟进或回访安排。
- **项目（Project）** — 与项目相关的里程碑或会议日程。

## 使用说明

### 创建日历事件

1. 进入日历页面（`/calendar`）。
2. 点击目标日期空白区域或点击「新建事件」按钮。
3. 填写事件标题和描述。
4. 选择事件类型（任务、会议、节假日、提醒）。
5. 设置开始时间和结束时间。
6. 若为全天事件，勾选「全天事件」选项。
7. 选择关联实体（可选）：客户、线索或项目。
8. 保存事件。

### 查看和管理事件

- 在周视图或月视图中，点击事件标题查看详情。
- 可直接在详情面板中编辑事件信息。
- 完成事件后，可标记为「已完成」。
- 支持拖拽调整事件时间（周视图和月视图）。

### 与工作台集成

日历事件会自动出现在工作台（Workbench）的"今日日程"小组件中，展示当天需要关注的所有日程，帮助您快速了解今日安排。

## 注意事项

- 全天事件的开始时间为当天 00:00:00，结束时间为当天 23:59:59。
- 任务和跟进记录创建后会自动生成日历事件，无需重复创建。
- 已完成的事件在日历中以特殊样式标记，与未完成事件区分。
