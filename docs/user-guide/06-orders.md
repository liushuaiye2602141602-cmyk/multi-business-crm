# 订单管理（Orders）

## 概述

订单模块是外贸业务流程的最终环节，用于管理从报价接受到订单交付的完整过程。订单模块支持完整的订单生命周期管理，包括订单创建、生产跟踪、发货管理和收款管理，并与发票（Invoice）系统集成，实现财务闭环。

## 订单状态流转

订单按以下流程推进：

```
DRAFT（草稿）
  ↓
CONFIRMED（已确认）
  ↓
PRODUCTION（生产中）
  ↓
READY_TO_SHIP（待发货）
  ↓
SHIPPED（已发货）
  ↓
COMPLETED（已完成）/ CANCELLED（已取消）
```

| 状态 | 中文 | 说明 |
|------|------|------|
| `DRAFT` | 草稿 | 订单已创建但尚未确认 |
| `CONFIRMED` | 已确认 | 订单已正式确认，进入执行阶段 |
| `PRODUCTION` | 生产中 | 产品正在生产制造 |
| `READY_TO_SHIP` | 待发货 | 产品已生产完成，等待发货 |
| `SHIPPED` | 已发货 | 产品已发出 |
| `COMPLETED` | 已完成 | 订单全流程完成（含收款确认） |
| `CANCELLED` | 已取消 | 订单已取消 |

## 操作步骤

### 新建订单

1. 导航至 `/orders/new`
2. 填写订单基本信息（标题）
3. 选择关联客户、项目和联系人
4. 选择币种，填写汇率
5. 填写商务条款（付款条件、交货条款、预计交货日期）
6. 关联来源报价单（如适用）
7. 点击"保存"完成创建（此时状态为 `DRAFT`）

### 添加订单项目

1. 在订单详情页，点击"添加项目"按钮
2. 填写产品名称、规格、数量、单价等信息
3. 可选择关联已有产品记录
4. 保存后，项目将按排序序号显示在订单明细列表中

### 确认订单

1. 确认订单内容和价格无误后，将状态变更为 `CONFIRMED`
2. **订单确认后会自动触发以下操作：**
   - 通过事件总线（`order.confirmed` 事件）自动创建生产跟进任务
   - 任务内容："跟进生产进度：{订单编号}"
   - 优先级：中（MEDIUM）
   - 截止时间：确认后 7 天的 09:00

### 生产跟踪

1. 订单确认后，状态变更为 `PRODUCTION`
2. 生产过程中通过自动创建的跟进任务定期检查生产进度
3. 生产完成后，状态变更为 `READY_TO_SHIP`

### 发货与完成

1. 产品发出后，将状态变更为 `SHIPPED`
2. 收到客户确认收货及完成所有收款后，将状态变更为 `COMPLETED`

## 发票系统（Invoice）

### 发票状态（InvoiceStatus）

## 已知限制

1. **发票管理**：当前版本部分支持发票的完整生命周期管理，部分发票操作（如自动生成发票）需要后续版本完善。
2. **订单变更**：订单进入 `PRODUCTION` 阶段后，修改订单内容需要谨慎操作，系统当前版本部分支持订单变更审批流程。
3. **发货追踪**：当前版本不支持对接第三方物流追踪系统，物流状态需手动更新。
4. **多币种结算**：订单确认后币种不可变更，如需变更币种需取消当前订单并重新创建。
5. **文件上传**：合同、装箱单等订单附件当前版本仅支持元数据记录，暂不支持实际文件上传，计划在后续版本实现。
6. **多租户数据隔离**：当前版本部分支持，计划在 v2.0 版本中全面实现。
